## Summary of Key UI Patterns

### Typography
- **Main headings:** Large, bold, dark text
- **Subheadings:** Medium size, gray text
- **Body text:** Regular weight, dark gray
- **Labels:** Uppercase, small, gray (e.g., "MENU", "MEMBERS")

### Color Scheme
- **Primary:** Orange/coral (#FF6B35 or similar) - buttons, badges, active states
- **Success:** Green - completed states, positive metrics
- **Error:** Red - failed states, alerts
- **Warning:** Yellow/amber - action items
- **Info:** Blue - email tags, some badges
- **Neutral:** Gray - inactive, disabled, borders
- **Background:** White cards on light gray/beige page background

### Button Styles
- **Primary:** Orange background, white text, rounded corners
- **Secondary:** White background, gray border, dark text
- **Icon buttons:** Circular or square, minimal styling

### Card Design
- White background
- Subtle shadow or border
- Rounded corners
- Padding around content
- Often with colored left border for status

### Badges
- Small, rounded rectangles
- Colored background or outline
- Uppercase or title case text
- Used for status, categories, counts

### Navigation
- Sidebar always visible (except workflow page)
- Icons + text labels
- Active state: highlighted background
- Collapsible sidebar option

### Member Avatars
- Circular
- Two-letter initials
- Colored backgrounds (orange, blue, purple, etc.)
- Consistent size

### Data Visualization
- Progress bars: Horizontal, colored (orange for progress)
- Metrics: Large numbers with small labels
- Charts: Horizontal bars with multiple series
- Status indicators: Colored dots, badges

### Form Elements
- Clean, minimal design
- Placeholder text in gray
- Icons inside input fields
- Dropdowns with chevron indicators
